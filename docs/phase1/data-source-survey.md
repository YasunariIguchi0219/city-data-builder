## 2. ソース別詳細（note.md指定の9項目）

### 2.1 Wikidata ✅

- **取得内容**：人口・座標・標高・国・行政区分・施設属性（P31）・世界遺産指定（P1435）・Wikipedia記事リンク（sitelinks、全言語版数も認知度指標に使える）
- **カバー率**：QID/座標100%、人口98%
- **取得方法**：REST API（wbgetentities）＋SPARQL（query.wikidata.org）。フェーズ0で取得パイプライン構築済み
- **再現性・更新頻度**：APIはリアルタイム。レート制限あり（実測：無認証は約1.5秒/件が安全）
- **商用利用・帰属・再配布**：CC0。制約なし。**他データと混ぜても汚染しない基盤ソース**
- **費用**：無料

### 2.2 Wikimedia Pageviews API ✅

- **取得内容**：記事別・月別の閲覧数（ja/en別に取れる。**日本語版閲覧数＝日本人の認知度**として使える）
- **実測例（2025年計）**：パリ ja 12.2万 / en 190万、ヴェルナッツァ ja 1,779 / en 5.2万 — 「有名すぎない穴場」判定に十分な解像度
- **カバー率**：ja 92.1% / en 98.0%（記事があるもののみ。ja欠損12件は英語版閲覧数で代替可）
- **取得方法**：REST API（per-article monthly）。User-Agentヘッダ必須
- **商用利用**：可（データはCC0）
- **費用**：無料
- **注意**：ja記事なし12件のうちスタバンゲル等はWikidataエンティティの粒度問題（都市圏エンティティにja記事が紐付いている等）の可能性。フェーズ3で個別確認

### 2.3 OurAirports ✅

- **取得内容**：空港の位置・種別（large/medium/small）・IATA/ICAOコード・定期便有無
- **実測**：欧州の定期便あり空港528件をダウンロードし、**全152地点の最寄り空港と距離を計算済み**（`data/raw/airports/nearest.json`）。距離中央値22km。100km超は4件のみ
- **取得方法**：CSV一括ダウンロード（毎晩更新）。取得スクリプト作成済み
- **商用利用・帰属・再配布**：パブリックドメイン。制約なし
- **費用**：無料

### 2.4 気候：Copernicus ERA5 ✅（CDS直接取得を採用）

- **取得内容**：note.md要求の4指標（月別気温・降水量・日照時間・湿度）すべて取得可能。座標ベースなので**全地点100%カバー**
- **実測**：ERA5ベースのAPIでパリ（7月平均19.8℃）・ツェルマット（7月13.1℃）の月別値を取得、妥当性確認済み
- **取得方法**：Copernicus CDS（要無料登録）からERA5月別統計を一括取得。平年値（例：直近10〜30年平均）を一度計算して保存する設計を想定
- **商用利用**：可。帰属表示必須：「Generated using Copernicus Climate Change Service information [年]」（改変時は「Contains modified …」）
- **費用**：無料
- **⚠️ 代替案の注意**：手軽なOpen-Meteo APIは**無料枠が非商用限定**。商用サービスで使うなら$29〜99/月の商用プラン契約が必要。平年値を一度計算して保存する用途ならERA5直接取得（無料・商用可）が合理的

### 2.5 世界遺産：UNESCO公式 ❌ → Wikidata経由 🔄

- **UNESCO公式（whc.unesco.org）を不採用とする理由**：
  1. 技術面：公式XMLはCloudflareで機械取得がブロックされる（実測）
  2. 法務面：シンジケーション規約により**商用利用は有償ライセンス＋事前書面許可が必要**。再公開は全面的に許可制
- **代替（実測済み）**：WikidataのP1435（heritage designation）=世界遺産で、**欧州域内1,076件（構成資産含む）を座標付きで取得成功**。登録の事実・座標・名称はCC0で利用可
- **注意**：UNESCOサイトの説明文は転載しない（事実データのみ使う）
- **設計課題**：都市と世界遺産の紐付けは距離だけでは不正確（例：バースは市自体が遺産）。Wikidata側の「所在地」プロパティ併用をフェーズ2で設計

### 2.6 POI：Foursquare OS Places ✅（第一候補）

- **取得内容**：全世界1億件超のPOI（名称・カテゴリ・座標）。飲食・文化・商業・ナイトライフ等のカテゴリ体系あり
- **ライセンス**：Apache 2.0。**share-alikeがなく他データと自由に混合できる**——ODbL制約のあるOSMに対する最大の優位点
- **取得方法**：Hugging Face（`foursquare/fsq-os-places`、要アカウント＋ゲート同意）／S3／Places Portal。月次更新
- **注意**：HF経由は「社名をFoursquareがマーケティングに使うことへの同意」がゲート条件。広報・法務に事前共有するか、S3/Portal経由での取得可否を確認
- **カバー率**：**未実測**（データが数GB規模のため本調査ではダウンロード未実施）。フェーズ3冒頭でサンプル都市（大・中・小の3件）についてパイロット計測を行い、特に小さな町・村での充足度を確認してから本採用を確定する
- **費用**：無料
- **決定（2026-07-16）：取得は保留し、フェーズ3のPOIはOSM/Overpassで先行整備する**。S3の匿名アクセスは実測の結果不可（ライセンス文書のみ公開）、Hugging Faceはゲート必須を確認（匿名401）。残る経路はHFゲート同意（広報マター）か Places Portal無料登録（規約要確認）で、どちらも後日の検討事項とする。poiブロックは `_meta.source="osm"` で整備し、Foursquare入手後に差し替え・比較検証する

### 2.7 OpenStreetMap ⚠️（自然地物・展望地点等に限定して条件付き採用）

- **取得内容**：POIに加え、**Foursquareにない旅行特有の地物**——展望地点(viewpoint)、ハイキングコース、海岸・湖・山、歩行者エリア、旧市街——が強み
- **実測**：Overpass APIでパリ半径5kmの飲食店15,676件の取得に成功（方式実証）。ただし**公開Overpassサーバーは過負荷が常態**（504多発）で、152地点×多カテゴリの一括取得には不向き
- **推奨取得方法**：Geofabrikの地域抽出ファイル（PBF）をダウンロードし、ローカルで集計（再現性・安定性とも公開APIより優る）
- **ライセンス（ODbL）の要点**（詳細は付録）：
  - 商用利用自体は可。帰属表示「© OpenStreetMap contributors」が必要
  - **社内DB内部利用はshare-alike対象外**（ODbL §4.5(c)に明文）が、「社内」の範囲（グループ会社・委託先・クラウド形態）は法務確認が必要
  - OSM由来データは**カラム単位で他ソースと分離**し、相互参照・相互補完をしない（Collective Database設計）。これを守れば派生DB扱いを回避できる
  - AI提案文はProduced Work（帰属のみ）と解釈できる材料があるが、LLM出力への公式解釈は存在しない
- **採用方針**：飲食店・文化施設など一般POIはFoursquare（Apache 2.0）を第一候補とし、OSMは**自然地物・展望地点・歩行者エリアなどFoursquareで代替できない層に限定**して使う。これによりODbL論点の影響範囲を最小化する
- **決定（2026-07-16）：社内利用か外部公開か未定のため「取得は進め、後からdisable可能」な設計とする**。ODbLの義務は公開時点で初めて発動するため、取得・社内加工の段階ではリスクの先取りにならない。disable可能にする条件として、(1) OSM由来データの物理分離（専用ブロック＋sourceメタデータ）、(2) 他ソースとの同一項目内での相互補完の禁止、(3) **派生指標への依存ソース記録**（OSM依存の指標をdisable時に機械的に特定・再計算するため）をフェーズ2のスキーマ設計に組み込む。OSMがNGになった場合に失うのはOSM固有層（展望地点・ハイキングコース・歩行者エリア等）のみで、一般POIはFoursquare＋Wikidataで維持される

### 2.8 Eurostat 観光統計 ⚠️

- **取得内容**：宿泊者数・宿泊施設数など（tour_occ系）。**NUTS2（広域地域）単位**——都市単位ではない
- **実測**：REST APIで2023年宿泊数の取得成功（アンダルシア7,390万泊、イル・ド・フランス8,516万泊）
- **制約**：
  1. 粒度：154件の大半は「都市＜NUTS2」なので、値は「その都市を含む地域の観光規模」としてしか使えない（認知度指標の補助として利用）
  2. カバー：ボスニア・ヘルツェゴビナ（サラエボ等3件）はEU統計域外
  3. ライセンス：CC BY 4.0だが、**非EU/EFTA由来データは商用再利用不可の例外**あり。使用する系列ごとに確認
- **必要な追加作業**：152地点→NUTS2コードの対応表作成（座標からNUTS境界データで機械判定可能）

### 2.9 外務省 海外安全情報 ✅

- **取得内容**：国・地域別の危険情報レベル（レベル1〜4）・感染症危険情報
- **実測**：オープンデータサイト（ezairyu.mofa.go.jp/html/opendata/）に**国別XMLが存在することを確認**（対象21カ国は全て提供範囲内）。なお欧州154件の大半は平時レベル0〜1想定
- **商用利用**：可（政府標準利用規約2.0、CC BY 4.0互換を明記）
- **帰属・加工表示**：「出典：外務省 海外安全情報オープンデータ（URL）」＋加工した旨の表示が必要。**「国が作成したかのような態様」での表示は禁止**——AI提案文に織り込む際のUI設計要件
- **粒度の限界**：国・地域単位。都市への展開は国レベル値の割り当てとし、粒度をメタデータに明記
- **費用**：無料。危険情報発出時に随時更新

### 2.10 都市間距離・空港距離 ✅（自前計算）

- **実測**：マスタの座標（CC0）から全地点の最寄り空港距離を計算済み。都市間距離行列（152×152）も同じ方法で即時生成可能
- **制約**：直線距離（大圏距離）。実移動時間・経路は将来課題（無料での経路APIは大量利用に制約が多い）
- **費用・ライセンス**：制約なし

### 2.11 GeoNames 🔷（補欠）

- CC BY 4.0、日次更新、人口・地名・行政区分。Wikidataで足りない場合の補完用として登録のみ。現時点で必須の用途なし

## 3. リスクと対応（ライセンス調査の結論）

| 順位 | 論点 | 対応 |
| --- | --- | --- |
| 1 | UNESCO公式は商用有償 | 不採用。Wikidata経由に切替済み（実測済み） |
| 2 | OSMのODbL | 利用範囲を自然地物等に限定＋カラム分離設計＋帰属表示。「社内利用」の範囲・AI出力の扱いは**法務確認を起票**（論点整理は付録に記載済み） |
| 3 | Open-Meteo無料枠は非商用限定 | 商用投入前にERA5直接取得へ切替（無料）または商用プラン契約 |
| 4 | Eurostatの非EU例外・粒度 | 使用系列の出所確認＋NUTS2粒度をメタデータに明記 |
| 5 | Foursquare HFゲート条件 | 広報・法務に事前共有。S3/Portal経由の代替も確認 |
| 6 | 外務省の加工表示義務 | 出典・加工表示のUI設計をglobal-bm側と調整 |
